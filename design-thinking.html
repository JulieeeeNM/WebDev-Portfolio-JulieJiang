Design Thinking Process

This page explains how I applied design thinking principles to create this portfolio website. Design thinking is a human-centered approach to problem solving that emphasizes empathy, ideation, and experimentation.

1

Empathize

I began by understanding my target audience - potential employers, clients, and collaborators. I considered:

  • What information would they be looking for?
  • How might they navigate the site?
  • What would make the experience enjoyable and efficient?

I also analyzed other portfolio websites to identify common patterns and opportunities for differentiation.

2

Define

Based on my research, I defined the key objectives for my portfolio:

  • Showcase my skills and projects effectively
  • Provide easy access to contact information
  • Demonstrate understanding of modern web practices
  • Ensure accessibility for all users
  • Reflect my personal brand and style

I created user personas and user stories to guide my design decisions.

3

Ideate

In this phase, I brainstormed multiple solutions for each aspect of the website:

  • Navigation structure alternatives
  • Layout options for project showcases
  • Color schemes and typography combinations
  • Interactive elements to engage visitors

I created rough sketches and wireframes to visualize different approaches.

4

Prototype

I developed interactive prototypes at different fidelity levels:

  1. Low-fidelity paper prototypes for quick testing
  2. Digital wireframes using Figma
  3. High-fidelity mockups with actual content
  4. Functional HTML/CSS prototypes

Each iteration incorporated feedback from peers and mentors.

5

Test

I conducted usability tests with 5 participants representing my target audience. Key findings included:

  • Users wanted quicker access to project details
  • Some color combinations affected readability
  • Navigation needed to be more intuitive on mobile

I addressed these issues in the final implementation, adding modal windows for projects, adjusting colors for better contrast, and improving the mobile menu.

Reflection

The design thinking process helped me create a more user-centered portfolio. Key takeaways:

  • Early user feedback prevented costly redesigns later
  • Multiple iterations led to better solutions
  • Documenting the process helped me make intentional decisions

Moving forward, I'll continue applying this approach to all my web projects.
